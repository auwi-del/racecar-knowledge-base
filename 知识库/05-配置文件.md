## 1. EKF 配置：`config/ekf.yaml`

robot_localization 的 EKF 节点参数。用于融合编码器轮速与 IMU 数据，输出平滑的里程计估计。

```yaml
ekf_filter_node:
  ros__parameters:
    frequency: 10.0              # EKF 运行频率 (Hz)
    sensor_timeout: 2.0          # 传感器超时时间 (s)
    two_d_mode: true             # 2D 模式（忽略高度变化）
    transform_timeout: 0.025     # TF 等待超时
    debug: false                 # 调试模式
    publish_tf: true             # 发布 TF 变换
    publish_acceleration: false  # 不发布加速度
    
    # 坐标系
    map_frame: map
    odom_frame: odom_combined    # 重要：里程计坐标系名
    base_link_frame: base_footprint
    world_frame: odom_combined
    
    # 输入1：编码器里程计 (/encoder)
    odom0: encoder               # 话题名
    odom0_config: [false, false, false,    # x, y, z 位置不用
                   false, false, false,    # roll, pitch, yaw 不用
                   true,  true,  false,    # vx, vy 用, vz 不用
                   false, false, true,     # vroll, vpitch 不用, vyaw 用
                   false, false, false]    # ax, ay, az 不用
    odom0_queue_size: 100
    odom0_differential: false
    odom0_relative: true
    
    # 输入2：IMU (/IMU_data)
    imu0: IMU_data
    imu0_config: [false, false, false,
                  false,  false,  true,    # 只使用 yaw（航向角）
                  false, false, false,
                  false,  false,  false,
                  false,  false,  false]
    imu0_differential: false
    imu0_relative: true
    imu0_queue_size: 100
    imu0_remove_gravitational_acceleration: true  # 去除重力加速度
    
    # 控制输入
    use_control: false           # 不使用 cmd_vel 作为预测输入
    control_config: [true, false, false, false, false, true]  # vx, vyaw
    
    # 运动限制
    acceleration_limits: [1.3, 0.0, 0.0, 0.0, 0.0, 3.4]
    deceleration_limits: [1.3, 0.0, 0.0, 0.0, 0.0, 4.5]
    
    # 噪声协方差矩阵 (15x15)
    process_noise_covariance: [0.5, 0.0, ...]  # 对角线：x/y=0.5, z=1.0, roll/pitch=0.03, yaw=0.06, vx/vy=0.025, vz=0.04, ...
```

**调参要点**：
- `odom0_relative: true` — 编码器数据作为相对里程计
- `imu0_remove_gravitational_acceleration: true` — IMU 需要去除重力
- `frequency` 从 10Hz 到 60Hz 有多种备选设置，当前主用 10Hz
- `odom_frame: odom_combined` — 与 Nav2 配置中的 odom_topic 一致

## 2. EKF Carto 配置：`config/ekf_carto.yaml`

用于 Cartographer SLAM 模式的 EKF 配置。主要区别：
- `frequency: 30.0` — 更高频率
- `publish_tf: false` — Cartographer 自己处理 TF
- `odom0: odom` — 使用不同话题
- 带完整中英文注释的模板

## 3. Nav2 全套配置：`config/nav.yaml`

包含以下所有 ROS2 参数命名空间的配置：

### AMCL 定位

```yaml
amcl:
  ros__parameters:
    base_frame_id: "base_footprint"
    global_frame_id: "map"
    odom_frame_id: "odom_combined"
    scan_topic: /scan
    laser_model_type: "likelihood_field"
    max_particles: 2000
    min_particles: 500
    pf_err: 0.05
    pf_z: 0.99
    update_min_a: 0.06    # 旋转更新阈值 (rad)
    update_min_d: 0.025   # 平移更新阈值 (m)
    set_initial_pose: True
    initial_pose.x: 0.0
    initial_pose.y: 0.0
    initial_pose.yaw: 0.0
    robot_model_type: "nav2_amcl::DifferentialMotionModel"
    sigma_hit: 0.2
    z_hit: 0.7
    z_rand: 0.059
    z_short: 0.24
```

### BT Navigator（行为树导航）

```yaml
bt_navigator:
  ros__parameters:
    global_frame: map
    robot_base_frame: base_footprint
    odom_topic: odom_combined
    goal_reached_tol: 0.6      # 到达目标的容差 (m)
    bt_loop_duration: 10       # BT 循环间隔 (ms)
    default_server_timeout: 20 # 动作服务器超时 (ms)
```

### Controller Server（控制器）

```yaml
controller_server:
  ros__parameters:
    controller_frequency: 20.0
    odom_topic: odom_combined
    controller_plugins: ["FollowPath"]
    FollowPath:
      plugin: "nav2_regulated_pure_pursuit_controller::RegulatedPurePursuitController"
      desired_linear_vel: 0.33          # 目标线速度 (m/s)
      lookahead_dist: 0.3               # 前视距离 (m)
      min_lookahead_dist: 0.3
      max_lookahead_dist: 0.6
      use_velocity_scaled_lookahead_dist: true
      use_regulated_linear_velocity_scaling: true  # 弯道减速
      regulated_linear_scaling_min_speed: 0.25     # 最低速度
      allow_reversing: false            # 不允许倒车
      use_collision_detection: false    # 不启用碰撞检测
```

### 代价地图

**局部代价地图** (rolling window, 4m×4m)：
```yaml
local_costmap:
  global_frame: odom_combined
  robot_base_frame: base_footprint
  rolling_window: true
  width: 4
  height: 4
  resolution: 0.05
  robot_radius: 0.60
  plugins: ["voxel_layer", "inflation_layer"]
```

**全局代价地图** (静态地图+障碍物+膨胀)：
```yaml
global_costmap:
  global_frame: map
  robot_base_frame: base_footprint
  resolution: 0.2
  track_unknown_space: true
  plugins: ["static_layer", "obstacle_layer", "inflation_layer"]
```

## 4. 激光雷达参数：`lslidar_driver/params/lsx10.yaml`

```yaml
lslidar_driver_node:
  ros__parameters:
    frame_id: laser_link
    device_ip: 192.168.1.200       # 雷达源 IP
    msop_port: 2368                # 数据端口
    difop_port: 2369               # 配置端口
    lidar_name: M10                # 雷达型号
    min_range: 0.4                 # 最小检测距离 (m)
    max_range: 200.0               # 最大检测距离 (m)
    scan_topic: /scan              # 发布的扫描话题
    interface_selection: serial    # 串口模式（非网口）
    serial_port_: /dev/laser       # 串口设备路径
    pubScan: true                  # 发布 /scan
    pubPointCloud2: false          # 不发布点云
    angle_disable_min: 0.0         # 角度裁剪（0=不裁剪）
    angle_disable_max: 0.0
```

## 8. Udev 规则文件

`racecar_init.sh` 会从当前目录复制以下 udev 规则文件：
- `car.rules` → 设置 `/dev/car`（底盘）
- `laser.rules` → 设置 `/dev/laser`（激光雷达）
- `imu.rules` → 设置 `/dev/imu`（IMU）
- `encoder.rules` → 设置 `/dev/encoder`（编码器）

这些规则文件确保每次插拔后设备路径固定。重新烧录系统或添加新设备后需要运行：
```bash
bash ~/racecar/racecar_init.sh
```
