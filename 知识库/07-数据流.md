## 关键数据链路详解

### 链路1：键盘遥控

```
键盘输入 → racecar_teleop.py
         → /teleop_cmd_vel (Twist)
         → racecar_driver_node::teleop_TwistCallback
         → serial → 下位机
```

**转换公式**：
```
angle = 2500.0 - twist.angular.z * 2000.0 / 180.0
linear_x = twist->linear.x   (直接透传)
send_cmd(linear_x, angle)
```

### 链路2：锥桶循迹（激光雷达）

```
LiDAR → /scan → lidar_tracking_node
                → 锥桶检测 (detectCones)
                → 数据关联 (associateCones)
                → 标准化 (standardizeCones)
                → 误差计算 (加权平均)
                → PID 控制 (PIDPositional)
                → /teleop_cmd_vel (Twist)
                → racecar_driver_node::teleop_TwistCallback
                → serial → 下位机
```

### 链路3：锥桶循迹（摄像头）

```
Camera → /image_raw → zhuitong.py
                      → HSV 阈值分割
                      → 锥桶检测、位置判断
                      → 生成转向指令
                      → /car_cmd_vel (Twist)
                      → racecar_driver_node::TwistCallback
                      → serial → 下位机
```

### 链路4：自主导航

```
用户设点 (RViz 2D Nav Goal) 或 waypoint_cycle
         → /goal_pose (PoseStamped)
         → Nav2 bt_navigator
           → planner_server → 全局路径 (/plan)
           → controller_server → 局部控制
         → /car_cmd_vel (Twist)
         → racecar_driver_node::TwistCallback
         → serial → 下位机
```

### 链路5：视觉巡线

```
Camera → /image_raw → line_follow.py
                      → 图像处理（灰度/二值化/线检测）
                      → 获取线偏移
                      → PD 控制
                      → /car_cmd_vel (Twist)
                      → racecar_driver_node::TwistCallback
                      → serial → 下位机
```

## 话题频率概览

| 话题 | 典型频率 | 用途 |
|------|----------|------|
| `/encoder` | 50Hz | 编码器轮速 |
| `/IMU_data` | ~100Hz | IMU 姿态 |
| `/scan` | ~10Hz | 激光雷达扫描 |
| `/odom_combined` | 10Hz (EKF) | 融合里程计 |
| `/tf` | 随传感器更新 | 坐标变换 |
| `/car_cmd_vel` | 可变 | 速度指令 |
| `/teleop_cmd_vel` | 可变 | 遥控速度指令 |
| `/image_raw` | ~30Hz | 摄像头画面 |
| `/goal_pose` | 按需 | 导航目标 |
| `/local_costmap/costmap_raw` | 10Hz | 局部代价地图 |
| `/global_costmap/costmap` | 10Hz | 全局代价地图 |
| `/plan` | 按需 | 规划路径 |

## 注意

- **ROS_DOMAIN_ID** 未设置（默认 0）。如果同一网络中有其他 ROS2 系统，需要设置唯一的 ROS_DOMAIN_ID 避免话题串扰。
- 话题命名中 `/odom_combined` 被所有节点引用，不要随意更改。如果修改 EKF 配置中的 `odom_frame`，需要同步修改 nav.yaml 中的 `amcl.odom_frame_id` 和 `controller_server.odom_topic`。
- `TwistCallback` 和 `teleop_TwistCallback` 使用不同的控制公式，注意区分。
