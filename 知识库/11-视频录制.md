# 摄像头视频录制

## 概述

davinci-mini 小车上有一个 USB 摄像头（`/dev/video0`），通过 ROS2 驱动（`usb_cam`）发布 `/image_raw` 话题。录制视频有三种方式，各有优劣。

---

## 方式一：ros2 bag record（推荐）

ROS2 自带的数据录制工具，录制为 `.db3` 数据库文件。**不影响小车运行，可同时录制任意数量话题。**

### 开始录制

```bash
# 录制摄像头话题（最常用）
ros2 bag record /image_raw

# 同时录制多个话题（摄像头 + 激光雷达 + 里程计）
ros2 bag record /image_raw /scan /odom_combined

# 录制所有话题
ros2 bag record -a

# 指定输出目录名
ros2 bag record -o my_recording /image_raw
```

### 停止录制

按 **Ctrl+C** 即可停止，文件自动保存。

### 录出来的文件在哪？

在当前目录生成一个文件夹，例如 `rosbag2_YYYY_MM_DD-HH_MM_SS/`，里面包含：
- `.db3` 文件 — SQLite3 数据库，存储所有消息数据
- `metadata.yaml` — 录制元信息（时长、话题列表等）

### 回放录制数据

```bash
ros2 bag play <bag目录>
```

回放后，你可以用 `rqt_image_view` 重新查看录制的画面。

### 压缩录制（节省空间）

```bash
ros2 bag record --compression-mode file --compression-format zstd /image_raw
```

## 方式二：使用 record.py（ROS 节点录制为 AVI）

已存在一个 ROS2 节点脚本 `~/racecar/src/racecar/scripts/record.py`，订阅 `/image_raw` 话题并保存为 AVI 文件。

### 开始录制

```bash
ros2 run racecar record.py
```

默认录制 **10 秒**后自动停止，输出文件为 `output.avi`。

### 修改录制时长

编辑 `record.py`，找到这一行修改数字：

```python
self.timer_thread = threading.Thread(target=self.stop_recording_after_delay, args=(10,))
```

把 `10` 改成你想要的秒数。

### 优缺点

| 优点 | 缺点 |
|------|------|
| 生成标准 AVI 文件，可直接播放 | 录制时长硬编码，需改代码 |
| 不占用摄像头设备 | 没有按键控制开始/停止 |
| 不干扰小车运行 | AVI 文件体积较大 |

---

## 方式三：使用 video_y.py（直接操作摄像头）

脚本 `~/video_y.py` 直接通过 OpenCV 打开 `/dev/video0`，有键盘交互。

### 开始录制

```bash
python3 ~/video_y.py
```

### 控制方式

| 按键 | 功能 |
|------|------|
| `f` | 开始录制（按一次即可） |
| `q` | 停止录制并退出 |

输出文件为 `video.mp4`。

### ⚠️ 重要限制

该脚本需要**独占** `/dev/video0` 设备。如果 `usb_cam` 节点正在运行（发布 `/image_raw`），则无法同时使用此脚本。**必须先停止摄像头节点：**

```bash
# 停止 usb_cam 节点
pkill -f usb_cam_node_exe

# 然后才能运行
python3 ~/video_y.py
```

---

## 与小车运行是否冲突？

| 录制方式 | 与导航/SLAM/巡线冲突？ | 原因 |
|----------|----------------------|------|
| `ros2 bag record /image_raw` | **不冲突 ✅** | 订阅话题而非操作硬件，ROS2 支持一话题多订阅者 |
| `record.py` | **不冲突 ✅** | 同样是订阅话题，与导航节点共存 |
| `video_y.py` | **冲突 ❌** | 直接打开 `/dev/video0`，会与 `usb_cam` 抢设备 |

### 结论

**小车在跑（自动巡航、建图、gmapping 等）的时候，完全可以同时录视频**，只要使用方式一（`ros2 bag`）或方式二（`record.py`）。因为 ROS2 的发布/订阅架构允许多个节点同时订阅同一个话题——摄像头节点只管拍照发布，录视频和跑导航都是各自订阅处理，互不干扰。

最推荐的组合：

```
小车跑导航 + 同时 ros2 bag record /image_raw /scan /odom_combined
```

这样不仅录了摄像头画面，还同时录了激光雷达和里程计数据，赛后可以完整回放分析。

## 实际录制测试数据

| 项目 | 值 |
|------|-----|
| 图像分辨率 | 640×480 |
| 帧率 | 30 FPS（实际约 15-30） |
| 录制 5 秒 bag 大小 | ~19 MB |
| 估算 1 分钟 bag 大小 | ~220 MB |
| 推荐录制时长 | 不超过 5 分钟（空间足够） |

**注意：** bag 文件存储在开发板上的 `~/` 或当前目录，录制完后可以用 `scp` 复制到 Windows 桌面：

```bash
# 在本机终端执行
scp -r davinci-mini@192.168.5.100:~/rosbag2_* /c/Users/LX/Desktop/
```
