# 启动系统

## 快速启动

所有启动操作均在 SSH 登录后执行，需要先 source 工作区：

```bash
cd ~/racecar
source install/setup.bash
```

但启动脚本（car.sh, nav.sh 等）内部已经处理了环境设置。

## 启动模式

本系统提供 **4 种启动模式**，通过 `~/racecar/` 下的 shell 脚本触发：

### 1. 仅底盘+传感器模式 (`car.sh`)

```bash
bash ~/racecar/car.sh
```

**启动内容**：
- EKF 里程计融合节点
- joint_state_publisher
- lslidar 激光雷达驱动 + rviz2
- HIPnuc IMU 发布节点
- encoder 编码器节点
- racecar_driver 底盘驱动
- 3 个静态 TF 发布器

**适用场景**：键盘遥控、锥桶循迹、视觉巡线

**调用链**：`car.sh` → `Run_car.launch.py` → 各个节点

### 2. 底盘+导航模式 (`nav.sh`)

```bash
bash ~/racecar/nav.sh
```

**启动内容**：
1. 先启动 `Run_car.launch.py`（同 car.sh）
2. 等待 10 秒
3. 再启动 `Run_nav.launch.py`
   - waypoint_cycle 航点循环节点
   - bringup_launch.py → AMCL 定位 + planner_server + bt_navigator

**适用场景**：已有地图的自主导航

### 3. 底盘+导航（one 版）(`nav_one.sh`)

```bash
bash ~/racecar/nav_one.sh
```

与 nav.sh 区别：使用 `Run_car_one.launch.py`，其中的 `racecar_driver` 订阅 `/cmd_vel` 而非 `/car_cmd_vel`

### 4. 底盘+Gmapping SLAM (`gmapping.sh`)

```bash
bash ~/racecar/gmapping.sh
```

**启动内容**：
1. 先启动 `Run_car.launch.py`
2. 等待 5 秒
3. 再启动 `slam_gmapping.launch.py`

**适用场景**：无地图时建图

### 5. 其他启动方式

**视觉巡线**：
```bash
source ~/racecar/install/setup.bash
ros2 launch racecar xunxian.launch.py
```

**单独启动摄像头**：
```bash
ros2 launch racecar camer.launch.py
```

## Launch 文件调用链

### Run_car.launch.py 完整调用链

```
Run_car.launch.py
├── ekf.launch.py
│   └── ekf_node (robot_localization)
│       ├── 条件: carto_slam=false → ekf_filter_node (使用 ekf.yaml)
│       └── 条件: carto_slam=true  → carto_ekf_filter_node (使用 ekf_carto.yaml)
├── joint_state_publisher
├── lslidar_launch.py (来自 lslidar_driver 包)
│   ├── lslidar_driver_node (LifecycleNode, 参数: lsx10.yaml)
│   └── rviz2 (启动 RViz)
├── imu_spec_msg.launch.py (来自 hipnuc_imu 包)
│   ├── IMU_publisher (talker, 参数: hipnuc_config.yaml)
│   └── imu_sub (listener)
├── encoder_node (encoder 包)
├── racecar_driver_node (racecar_driver 包)
│   ├── 订阅 /car_cmd_vel (TwistCallback)
│   └── 订阅 /teleop_cmd_vel (teleop_TwistCallback)
├── static_transform_publisher (base_footprint→base_link)
├── static_transform_publisher (base_footprint→laser_link)
└── static_transform_publisher (base_footprint→IMU_link)
```

### Run_nav.launch.py 完整调用链

```
Run_nav.launch.py
├── waypoint_cycle (nav2_waypoint_cycle)
│   ├── 发布 /path_point (MarkerArray)
│   ├── 发布 /goal_pose (PoseStamped)
│   └── 订阅 navigate_to_pose/_action/status
└── bringup_launch.py (来自 racecar 包, 封装 nav2_bringup)
    ├── slam_launch.py (条件: slam=true)
    ├── localization_launch.py (条件: slam=false) ← 默认走这里
    │   └── AMCL 定位节点
    └── navigation_launch.py
        ├── planner_server (SmacPlannerHybrid)
        └── bt_navigator (行为树导航)
```

注意：bringup_launch.py 自定义设置了 lifecycle_nodes 列表，只包含 `planner_server` 和 `bt_navigator`。controller_server, smoother_server, behavior_server 等节点**未被包含**，而是通过 nav.yaml 中的配置启动。

### Run_car_one.launch.py 与 Run_car.launch.py 的区别

唯一区别：
- `Run_car.launch.py` → 使用 `racecar_driver_node`（订阅 `/car_cmd_vel`）
- `Run_car_one.launch.py` → 使用 `racecar_driver_node_one`（订阅 `/cmd_vel`）

## 启动顺序注意事项

1. **nav.sh / nav_one.sh / gmapping.sh** 中，底盘启动后有 sleep 等待传感器初始化，再启动导航/SLAM
2. 如果 Nav2 启动时 `ai_map.yaml` 不存在，map_server 会报错。需要先用 gmapping 建图或准备地图文件
3. `lslidar_launch.py` 会同时启动 rviz2，如果不需要可在启动后关闭 rviz2 窗口

## 停止系统

按 `Ctrl+C` 终止所有后台进程。脚本中的 `trap terminate SIGINT` 会依次杀死各进程。
