# 节点架构

本系统是一个典型的 ROS2 机器人系统，节点按照**传感器 → 数据融合 → 规划控制 → 执行器** 的链路组织。

## 运行时节点（正常行驶模式）

以下是根据 `Run_car.launch.py` + `Run_nav.launch.py` 启动后的节点列表：

| 节点名 | 包 | 可执行文件 | 功能 |
|--------|-----|-----------|------|
| `ekf_filter_node` | robot_localization | `ekf_node` | EKF 传感器融合（编码器+IMU → 里程计） |
| `joint_state_publisher` | joint_state_publisher | `joint_state_publisher` | 关节状态发布（无实际关节，占位） |
| `lslidar_driver_node` | lslidar_driver | `lslidar_driver_node` | 激光雷达驱动（LifecycleNode） |
| `rviz2` | rviz2 | `rviz2` | 可视化 |
| `IMU_publisher` | hipnuc_imu | `talker` | IMU 数据发布 |
| `imu_sub` | hipnuc_imu | `listener` | IMU 数据订阅（日志用） |
| `encoder_vel` | encoder | `encoder_node` | 编码器数据读取与 Odometry 发布 |
| `racecar_driver` | racecar_driver | `racecar_driver_node` | 底盘驱动（Twist → PWM） |
| `base_footprint2base_link` | tf2_ros | `static_transform_publisher` | 静态 TF 发布 |
| `base_link2laser_link` | tf2_ros | `static_transform_publisher` | 静态 TF 发布 |
| `base_link2imu` | tf2_ros | `static_transform_publisher` | 静态 TF 发布 |
| `waypoint_cycle` | nav2_waypoint_cycle | `nav2_waypoint_cycle` | 航点循环（nav 模式） |
| 多个 Nav2 内部节点 | nav2_* | 多个 | AMCL 定位、规划器、控制器 |

## 话题通信全景

### 传感器话题（发布者）

| 话题名 | 类型 | 发布节点 | 频率 |
|--------|------|----------|------|
| `/scan` | sensor_msgs/LaserScan | `lslidar_driver_node` | ~10Hz |
| `/IMU_data` | sensor_msgs/Imu | `IMU_publisher` | ~100Hz |
| `/encoder` | nav_msgs/Odometry | `encoder_vel` | 50Hz |

### 融合/状态话题

| 话题名 | 类型 | 发布节点 | 说明 |
|--------|------|----------|------|
| `/odom_combined` | nav_msgs/Odometry | `ekf_filter_node` | EKF 融合后的里程计 |
| `/tf` | tf2_msgs/TFMessage | 多节点 | 坐标变换树 |
| `/tf_static` | tf2_msgs/TFMessage | 静态 TF 发布器 | 静态坐标变换 |

### 控制指令话题（订阅者）

| 话题名 | 类型 | 订阅节点 | 用途 |
|--------|------|----------|------|
| `/car_cmd_vel` | geometry_msgs/Twist | `racecar_driver` | 导航/程序控制速度指令 |
| `/teleop_cmd_vel` | geometry_msgs/Twist | `racecar_driver`, `lidar_tracking_node` | 键盘遥控/锥桶循迹速度指令 |
| `/cmd_vel` | geometry_msgs/Twist | `racecar_driver_node_one` | 旧版接口 |

### 控制指令话题（发布者）

| 话题名 | 类型 | 发布节点 | 用途 |
|--------|------|----------|------|
| `/car_cmd_vel` | Twist | `line_follow`, `zhuitong`, 等脚本 | 程序控制 |
| `/teleop_cmd_vel` | Twist | `racecar_teleop`, `lidar_tracking_node` | 遥控/循迹 |

### Nav2 内部话题

| 话题名 | 类型 | 用途 |

## 坐标变换树（TF）

```
map (仅在导航时)
 └─ odom_combined (EKF里程计)
     └─ base_footprint (机器人足印)
         ├─ base_link (z=0.15)
         │   传感器数据在此坐标系下
         ├─ laser_link (x=0.07)
         └─ IMU_link (x=0.1653)
```

## 生命周期节点

`lslidar_driver_node` 使用 ROS2 LifecycleNode 管理：

1. **Unconfigured** → `configure()` → **Inactive**
2. **Inactive** → `activate()` → **Active**（开始发布 /scan）
3. **Active** → `deactivate()` → **Inactive**
4. **Active** / **Inactive** → `cleanup()` → **Unconfigured**
5. **任何状态** → `shutdown()` → **Finalized**

Nav2 节点（controller_server, planner_server, bt_navigator 等）也使用生命周期管理，通过 `nav2_lifecycle_manager` 统一管理状态转换。
