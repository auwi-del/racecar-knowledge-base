# 操作指南

## 快速入门

### 1. 连接开发板

```bash
# 确保 PC 与开发板在同一个网段（192.168.5.x）
ssh davinci-mini@192.168.5.100
```

### 2. 初始化（仅首次或更换硬件后）

```bash
cd ~/racecar
bash racecar_init.sh
# 需要 sudo 权限，输入密码
```

### 3. 启动机器人

```bash
# 仅底盘+传感器（用于遥控/循迹）
bash ~/racecar/car.sh

# 或底盘+导航（用于自主导航）
bash ~/racecar/nav.sh
```

### 4. 控制机器人

**新开一个 SSH 终端**：

```bash
ssh davinci-mini@192.168.5.100
source ~/racecar/install/setup.bash

# 键盘控制
ros2 run racecar racecar_teleop
```

## 参数调整指南

### 临时调整（运行时）

```bash
# 查看运行时可调参数
ros2 param list

# 获取参数当前值
ros2 param get /ekf_filter_node frequency

# 设置参数（重启后失效）
ros2 param set /ekf_filter_node frequency 20.0

# 使用 rqt_reconfigure 可视化调参
ros2 run rqt_reconfigure rqt_reconfigure
```

### 永久调整

编辑对应 YAML 文件后重构：

```bash
cd ~/racecar
colcon build --packages-select racecar
source install/setup.bash
```

如果只改配置文件不涉及代码，也可以直接修改 `install/` 目录下的对应文件，无需编译：

```bash
# 直接修改安装目录的配置文件
nano ~/racecar/install/racecar/share/racecar/config/ekf.yaml

# 重启节点生效
```

### 重要参数速查

| 要调什么 | 改哪里 | 参数名 |
|----------|--------|--------|
| 行驶速度 | nav.yaml | controller_server.FollowPath.desired_linear_vel |
| 转弯灵敏度 | nav.yaml | controller_server.FollowPath.lookahead_dist |
| EKF 融合强度 | ekf.yaml | process_noise_covariance（对角线值） |
| 锥桶循迹灵敏度 | params.yaml (lidar_tracking) | kp, ki, kd |
| 锥桶循迹速度 | params.yaml (lidar_tracking) | speed |
| AMCL 定位精度 | nav.yaml | min_particles, max_particles |
| 避障距离 | nav.yaml | costmap.inflation_radius |
| 激光雷达范围 | lsx10.yaml | min_range, max_range |
| 编码器系数 | encoder_node.py (代码内) | k 参数, C 常数 |

## 常见问题排查

### 连接问题

| 问题 | 可能原因 | 解决 |
|------|----------|------|
| SSH 连不上 192.168.5.100 | 网线未接 / 不在同一网段 | 检查本机 IP：`ipconfig`；确保在 192.168.5.x 段 |
| SSH 连上但 ping 不通 | 防火墙 | 检查 `ufw status` |
| 串口设备找不到 | udev 规则未配置 | 运行 `bash ~/racecar/racecar_init.sh` |

### 运行问题

| 问题 | 可能原因 | 解决 |
|------|----------|------|
| 启动 car.sh 后没反应 | 串口设备未连接或权限不足 | 检查 /dev/car, /dev/laser, /dev/imu, /dev/encoder 是否存在 |
| robot_localization 报错 | EKF 配置参数错误 | 检查 ekf.yaml 格式，注意 ROS2 YAML 的 `ros__parameters:` 层级 |
| 激光雷达无 /scan 数据 | 设备未连接或参数错误 | 检查 /dev/laser，检查雷达 IP 和设备型号 |
| IMU 无数据 | 串口参数不匹配 | 检查 hipnuc_config.yaml 中的端口和波特率 |
| Nav2 不规划路径 | 地图未加载或 TF 树断裂 | 查看 map_server 是否正常运行，检查 TF 树 |
| 导航无法到达目标 | 代价地图参数不合适 | 调整膨胀半径、控制器参数 |
| 锥桶循迹车辆晃动 | PID 参数不合适 | 调整 kp/kd，检查车道宽度参数 |
| 摄像头画面无显示 | 设备不兼容或格式错误 | 检查 /dev/video0，尝试修改 pixel_format |

### 编译问题

```bash
# 缺少依赖
rosdep install -i --from-path src --rosdistro humble -y

# 单包编译
colcon build --packages-select <package_name>

# 清理并重编
rm -rf build/ install/ log/
colcon build
```

## 安全须知

1. **第一次运行时拿高车辆**，确保轮子离地，测试控制信号是否正常
2. **串口设备路径** (`/dev/car`, `/dev/laser`, `/dev/imu`, `/dev/encoder`) 依赖于 udev 规则，重新烧录系统后必须运行 `racecar_init.sh`
3. **电池电量** 不足时舵机/电机可能表现异常，注意充电
4. **激光雷达** 是旋转机械结构，确保其旋转无障碍物
5. **启动 nav.sh 时**，确保 `src/racecar/map/ai_map.yaml` 存在，否则 map_server 会报错
6. **多终端操作**：car.sh / nav.sh 会直接在前台运行，需要新开 SSH 终端运行其他的命令（控制、调试等）
