# Nav2 导航调参指南（nav.sh + go.py）

## 适用范围

本文档适用于 `bash ~/racecar/nav.sh` + `ros2 run racecar go.py` 链路中的导航行为调优。该链路使用 Nav2 导航栈 + Regulated Pure Pursuit 控制器，**不涉及 lidar_tracking 锥桶循迹的 PID 参数**。

## 数据流

```
Nav2 规划器 (SmacPlannerHybrid)
  → Nav2 控制器 (RegulatedPurePursuit)
  → /cmd_vel
  → go.py（转发）
  → /car_cmd_vel
  → racecar_driver_node（PWM 转换）
  → 底盘串口
```

角度控制完全由 Nav2 的 Regulated Pure Pursuit 控制器负责，**与 lidar_tracking/config/params.yaml（kp/ki/kd）毫无关系**。

## 典型问题与根因

### 1. 走起来左右晃（晃来晃去）

**根因**：前视距离（lookahead_dist）远小于最小转弯半径。

| 当前值 | 问题 |
|--------|------|
| `lookahead_dist: 0.3` | 前视仅 0.3m |
| `minimum_turning_radius: 0.6` | 最小转弯半径 0.6m |

车还没"看到"弯道就已经到弯里了，只能走一步看一步，频繁修正方向 → 左右晃动。

**调整**：
```yaml
lookahead_dist: 0.3  → 0.7       # 基础前视距离
min_lookahead_dist: 0.3 → 0.5    # 最小前视距离
max_lookahead_dist: 0.6 → 1.0    # 最大前视距离
```

原理：前视距离越大，路径跟踪越平滑，但过大会切弯；前视距离越小，转向越激进，容易震荡。

### 2. 撞墙（创墙）

**根因 1**：碰撞检测未开启。

```yaml
use_collision_detection: false    # ← 控制器完全不检查是否会撞
```

**调整**：
```yaml
use_collision_detection: true
max_allowed_time_to_collision_up_to_carrot: 2.0  # 留足反应时间
```

**根因 2**：膨胀半径太小，路径贴着墙规划。

| 代价地图 | 当前值 | 建议值 |
|----------|--------|--------|
| 全局膨胀半径 | `0.15` | `0.4` |
| 局部膨胀半径 | `0.25` | `0.5` |

车半径 0.6m，膨胀仅 15-25cm，规划路径几乎贴着墙。

**根因 3**：局部代价地图障碍物检测范围偏小。

```yaml
obstacle_max_range: 2.5  → 4.0   # 激光雷达障碍物检测范围
```

### 3. 走走停停（犹犹豫豫）

**根因**：参数矛盾 + 检测范围不足。

```yaml
min_approach_linear_velocity: 0.5   # 接近终点最低速度（错误！）
desired_linear_vel: 0.33            # 目标速度
```

`min_approach_linear_velocity` 比 `desired_linear_vel` 还高，接近终点时减速逻辑无法正常工作，导致机器人行为异常。

**调整**：
```yaml
min_approach_linear_velocity: 0.5 → 0.15
```

## 完整调整方案（按优先级）

### 一级调整（解决 80% 问题）

```yaml
controller_server.FollowPath:
  lookahead_dist: 0.7               # 0.3 → 0.7，增大前视
  min_lookahead_dist: 0.5           # 0.3 → 0.5
  max_lookahead_dist: 1.0           # 0.6 → 1.0
  use_collision_detection: true     # false → true，开启碰撞检测
  max_allowed_time_to_collision_up_to_carrot: 2.0
  min_approach_linear_velocity: 0.15  # 0.5 → 0.15，修复矛盾

local_costmap.inflation_layer:
  inflation_radius: 0.5             # 0.25 → 0.5

global_costmap.inflation_layer:
  inflation_radius: 0.4             # 0.15 → 0.4

local_costmap.voxel_layer.scan:
  obstacle_max_range: 4.0           # 2.5 → 4.0
```

### 二级调整（进一步优化）

如果一级调整后仍然不如意，可以尝试：

```yaml
# 降低速度，牺牲速度换稳定性
controller_server.FollowPath.desired_linear_vel: 0.28  # 0.33 →

# 增加规划器平滑
planner_server.GridBased.smoother:
  w_smooth: 0.5 → 0.7       # 更平滑的路径
  w_data: 0.1 → 0.05        # 更少保留原始路径形状

# 增加规划器的障碍物代价权重
planner_server.GridBased.cost_penalty: 1.7 → 2.5   # 让路径更偏向通道中央
planner_server.GridBased.cost_travel_multiplier: 2.0 → 3.0

# 增大行为服务器旋转加速度限制（让转头更柔顺）
behavior_server.rotational_acc_lim: 0.3 → 0.5
behavior_server.max_rotational_vel: 1.0 → 0.8
```

### 三级调整（高级）

```yaml
# 启用原地旋转对准航向（可消除大角度转弯时的犹豫）
controller_server.FollowPath.use_rotate_to_heading: false → true
controller_server.FollowPath.rotate_to_heading_min_angle: 0.785  # 约 45°
controller_server.FollowPath.rotate_to_heading_angular_vel: 0.4   # 旋转速度

# 如果仍然震荡，可以关闭速度缩放的前视（让前视距离恒定）
controller_server.FollowPath.use_velocity_scaled_lookahead_dist: true → false
```

## 运行时调参方法

### 修改单个参数测试（推荐）

```bash
ssh davinci-mini@192.168.5.100
source ~/racecar/install/setup.bash

# 查看当前值
ros2 param get /controller_server FollowPath.lookahead_dist

# 修改
ros2 param set /controller_server FollowPath.lookahead_dist 0.7
```

### 查看所有可调参数

```bash
ros2 param list /controller_server
ros2 param list /planner_server
```

### 可视化调参

```bash
ros2 run rqt_reconfigure rqt_reconfigure
```

### 永久修改

```bash
nano ~/racecar/src/racecar/config/nav.yaml
# 修改后重启 nav.sh 即可，不需要 colcon build（仅改配置文件不涉及代码时）
```

| 想解决什么 | 改哪里 | 参数 | 建议方向 |
|-----------|--------|------|---------|

## 注意事项

- **本文不涉及** lidar_tracking 锥桶循迹的 PID 参数（kp/ki/kd），那套参数在 `ros2 launch lidar_tracking lidar_tracking.launch.py` 时生效
- **本文不涉及** racecar_driver_node 中的角速度→PWM 缩放系数，那个系数影响所有模式下的转向幅度，但不是 PID 控制
- 调整膨胀半径时，确保 `cost_scaling_dist ≤ inflation_radius`，否则代价计算会异常
- 每次只改 1-2 个参数测试，不要一次性全改，否则无法判断哪个参数有效
