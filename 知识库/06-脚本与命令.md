# 脚本和命令

## 常用 ros2 CLI 命令

### 查看系统状态

```bash
# 查看所有节点
ros2 node list

# 查看节点信息
ros2 node info <node_name>

# 查看所有话题
ros2 topic list

# 查看话题类型和发布/订阅者
ros2 topic info <topic_name>

# 实时查看话题数据
ros2 topic echo <topic_name>

# 查看话题频率
ros2 topic hz <topic_name>

# 查看所有服务
ros2 service list

# 查看所有动作
ros2 action list

# 查看参数
ros2 param list
ros2 param get <node_name> <param_name>

# 查看 TF 树
ros2 run tf2_tools view_frames.py
# 或
ros2 topic echo /tf
```

### 控制命令

```bash
# 键盘遥操作
ros2 run racecar racecar_teleop

# 发布 Twist 指令（手动测试）
ros2 topic pub /car_cmd_vel geometry_msgs/Twist "{linear: {x: 0.5}, angular: {z: 0.0}}"

# 发布 Nav2 导航目标
ros2 topic pub /goal_pose geometry_msgs/PoseStamped "{header: {frame_id: 'map'}, pose: {position: {x: 1.0, y: 0.0}, orientation: {z: 0.0, w: 1.0}}}"

# 设置 AMCL 初始位姿
ros2 topic pub /initialpose geometry_msgs/PoseWithCovarianceStamped "{header: {frame_id: 'map'}, pose: {pose: {position: {x: 0.0, y: 0.0}, orientation: {z: 0.0, w: 1.0}}}}"
```

### 建图与导航

```bash
# 保存地图到指定路径
ros2 run nav2_map_server map_saver_cli -f ~/racecar/src/racecar/map/ai_map

# 或使用 save.sh 脚本（自动保存到 ai_map）
bash ~/racecar/save.sh

# 启动 Nav2 导航（如在 car.sh 运行后单独启动）
ros2 launch racecar Run_nav.launch.py

# 启动 SLAM Gmapping
ros2 launch slam_gmapping slam_gmapping.launch.py
```

### 调试与可视化

```bash
# 启动 RViz2
ros2 run rviz2 rviz2

# 参数动态配置
ros2 run rqt_reconfigure rqt_reconfigure

# 可视化节点/话题图
ros2 run rqt_graph rqt_graph

# 查看控制台日志
ros2 run rqt_console rqt_console

# 录制 rosbag
ros2 bag record -a -o my_bag

# 回放 rosbag
ros2 bag play my_bag

# 查看 TF 树（图形化）
ros2 run tf2_tools view_frames.py
evince frames.pdf  # 查看生成的 PDF
```

### 诊断与检查

```bash
# ROS2 系统诊断
ros2 doctor

# 查看接口类型定义
ros2 interface show sensor_msgs/msg/LaserScan
ros2 interface show geometry_msgs/msg/Twist

# 查看包信息
ros2 pkg list | grep <keyword>
ros2 pkg executables <package_name>
```

## Python 脚本详解

所有脚本位于 `~/racecar/src/racecar/scripts/`，需要通过以下方式运行：

```bash
# 方式1：先 source 再直接运行
source ~/racecar/install/setup.bash
ros2 run racecar <script_name_without_extension>

# 方式2：直接运行（需要脚本内有正确 shebang）
python3 ~/racecar/src/racecar/scripts/<script_name>.py
```

### 1. 键盘遥操作：`racecar_teleop.py`

**启动命令**：
```bash
ros2 run racecar racecar_teleop
```

**功能**：键盘控制机器人移动。

**按键说明**：
```
   u    i    o
   j    k    l
   m    ,    .

   i = 前进    , = 后退    j = 左转    l = 右转
   k / space = 停止
   w = 加速    x = 减速
   a = 左偏中  d = 右偏中
   Ctrl+C = 退出
```

**发布话题**：`/teleop_cmd_vel` (geometry_msgs/Twist)

### 2. 锥桶追踪（摄像头）：`zhuitong.py`

**启动命令**：
```bash
ros2 run racecar zhuitong
```

**功能**：通过 USB 摄像头识别红蓝锥桶，根据 HSV 颜色阈值检测并控制车辆行驶。

**关键参数**（代码内硬编码）：
- 蓝色 HSV：`[90, 110, 28]` ~ `[170, 255, 255]`
- 红色 HSV：`[50, 100, 60]` ~ `[70, 255, 255]`

**发布话题**：`/car_cmd_vel`

**特点**：包含时间触发的固定动作序列（例如 15~19秒时自动左转）

### 3. 视觉巡线：`line_follow.py` / `line_follow2.py`

**启动命令**：
```bash
# 方式1：通过 launch 文件
ros2 launch racecar xunxian.launch.py

# 方式2：直接运行
ros2 run racecar line_follow
```

**功能**：摄像头识别地面引导线，PID 控制沿线路行驶。

**xunxian.launch.py 启动内容**：
1. `camer.launch.py` → usb_cam 摄像头节点（320×240, yuyv）
2. `line_follow` 节点

### 4. 其他脚本

| 脚本 | 功能 | 备注 |
|------|------|------|
| `camer.py` | 显示摄像头画面 | 调试用 |
| `detct.py` | 目标检测 | OpenCV 实现 |
| `hsv_show.py` | HSV 颜色阈值调参 | 调试工具 |
| `record.py` | 数据记录 | 可扩展 |
| `go.py` / `get.py` | 测试动作 | 功能待确认 |
| `py_node.py` | ROS2 Python 节点模板 | 参考代码 |
| `ai_*.py` | AI 比赛脚本 | 比赛专用 |
| `red_light_detected.jpg` | 参考图片 | 静态资源 |

## 系统脚本

### `car.sh` / `nav.sh` / `nav_one.sh` / `gmapping.sh`

位于 `~/racecar/` 下的这些脚本是**主要启动入口**，封装了 ros2 launch 命令。

```bash
# 所有脚本的共同特性
# 1. 使用 trap 机制处理 Ctrl+C 优雅退出
# 2. 后台 (&) 启动 launch 进程
# 3. wait 等待所有进程结束
trap terminate SIGINT  # 捕获 Ctrl+C
```

### `save.sh`

```bash
#!/bin/bash
MAP_DIR="/home/davinci-mini/racecar/src/racecar/map"
MAP_NAME="ai_map"
ros2 run nav2_map_server map_saver_cli -f "$MAP_DIR/$MAP_NAME"
```

### `racecar_init.sh`

```bash
#!/bin/bash
sudo cp ./car.rules /etc/udev/rules.d
sudo cp ./laser.rules /etc/udev/rules.d
sudo cp ./imu.rules /etc/udev/rules.d
sudo cp ./encoder.rules /etc/udev/rules.d
```

## 构建命令

```bash
# 构建整个工作区
cd ~/racecar
colcon build

# 构建单个包
colcon build --packages-select <package_name>

# 构建并忽略测试
colcon build --cmake-args -DBUILD_TESTING=OFF

# 只构建依赖变更的包
colcon build --packages-up-to <package_name>
```
