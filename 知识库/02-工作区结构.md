### racecar_driver

底盘控制驱动。接收 Twist 速度指令，通过串口发送 PWM 信号给下位机。

**关键文件**：
- `src/racecar_driver_node.cpp` — 订阅 `/car_cmd_vel`（导航用）
- `src/racecar_driver_node_one.cpp` — 订阅 `/cmd_vel`（旧版接口）
- `include/racecar_driver.h` — 串口初始化与指令发送接口
- `lib/libracecar_driver.a` — 预编译的底层驱动库

**代码逻辑**：
1. 初始化串口连接到 `/dev/car`，波特率 38400
2. 订阅两个话题：`/car_cmd_vel` 和 `/teleocmd_vel`
3. TwistCallback：`linear_x = twist->linear.x * 100 + 1500; angle = 1500.0 - twist->angular.z * 1300`
4. teleop_TwistCallback：`angle = 2500.0 - twist->angular.z * 2000.0 / 180.0; linear_x = twist->linear.x`
5. 通过 `send_cmd(motor_pwm, servo_pwm)` 发送电机/舵机 PWM 值

### encoder

编码器节点（纯 Python）。通过串口读取磁编码器数据，计算速度并发布 Odometry 消息。

**关键文件**：
- `encoder/encoder_node.py` — 主节点
- `encoder/encoder_imu.py` — 编码器+IMU 融合版本（未使用）

**代码逻辑**：
1. 打开串口 `/dev/encoder`，波特率 57600
2. 发送 0x01 0x03 0x00 0x03 0x00 0x01 0x74 0x0A 读取速度
3. 发送 0x01 0x03 0x00 0x00 0x00 0x01 0x84 0x0A 读取方向/圈数
4. 计算：`Vel = 0.714 * angle_v * C / 1024.0 / 0.02 * k * 0.25 * 1.03`
5. 以 50Hz 发布 nav_msgs/Odometry 到 `/encoder` 话题

### hipnuc_imu

HIPnuc 品牌 IMU 的 ROS2 驱动。C++ 实现，通过串口读取九轴数据发布 sensor_msgs/Imu。

**关键文件**：
- `src/serial_port.cpp` — `IMUPublisher` 节点，读取串口并发布 `/IMU_data`
- `src/sub_spec.cpp` — `imu_sub` 节点，订阅 IMU 并打印日志
- `src/ch_serial.c` — 底层串口操作
- `config/hipnuc_config.yaml` — 参数：端口 `/dev/imu`，波特率 115200，帧 ID `IMU_link`

### lslidar_driver

LS LIDAR M10 激光雷达驱动。使用 LifecycleNode 实现生命周期管理。

**关键文件**：
- `params/lsx10.yaml` — 雷达参数：IP 192.168.1.200，MSOP 端口 2368，DIFOP 端口 2369
- 型号选择：`M10`（可选 M10_P, M10_PLUS, N10, L10 等）
- **注意**：激光雷达通过串口连接（`serial_port_: /dev/laser`），非网口

### lslidar_msgs

LS LIDAR 自定义消息类型包。定义了雷达专用的 ROS2 消息格式。

### lidar_tracking

核心循迹节点 — 使用激光雷达检测红蓝锥桶并控制车辆自动行驶。

**关键文件**：
- `src/lidar_tracking.cpp` — 主程序
- `config/params.yaml` — 可调参数

**功能**：详见 [08-源码参考.md](08-源码参考.md)。

### nav2_waypoint_cycle

Nav2 航点循环节点。用户在 RViz 中点击设置多个航点，机器人依次访问并自动循环。

**关键文件**：
- `nav2_waypoint_cycle/waypoint_cycle.py` — 主节点

**功能**：
1. 订阅 `/clicked_point`（RViz 的 PointStamped）
2. 在 RViz 中以箭头和编号显示已设置的航点
3. 通过 `/goal_pose` 逐个发布导航目标
4. 监听 `navigate_to_pose/_action/status` 判断是否到达
5. 到达后自动发下一个，全部完成则循环

### openslam_gmapping / slam_gmapping

Gmapping SLAM 算法的 ROS2 移植。openslam_gmapping 是核心算法库，slam_gmapping 是 ROS2 节点封装。

### serial-foxy

跨平台串口通信库（从 ROS1/foxy 移植），提供 C++ 串口读写接口。被 racecar_driver 直接或间接使用。
