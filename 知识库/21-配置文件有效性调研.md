# 配置文件有效性调研报告 — 哪些配置文件是死代码？

## 调研方法

1. 遍历 `src/racecar/config/` 和 `install/racecar/share/racecar/config/` 所有配置文件
2. 搜索全部 11 个自定义包的 launch 文件、C++ 源码、Python 脚本中对这些配置文件的引用
3. 交叉比对 `install/` 与 `src/` 中的重复副本
4. 追踪启动脚本调用链确认真实加载路径

---

## 一、配置文件分布总览

### 1.1 配置文件的物理位置（4 个副本层级）

| 层级 | 路径 | 用途 |
|------|------|------|
| 源码 | `src/racecar/config/` | 编辑修改的目标位置 |
| 构建副本 A | `install/racecar/share/racecar/config/` | **运行时实际加载位置**（launch 文件通过 `get_package_share_directory()` 指向此处） |
| 构建副本 B | `install/racecar/share/config/` | ❌ 幽灵目录 — 不属于任何 ROS 包，无任何代码引用 |
| 构建副本 (build) | `build/racecar/` | 构建中间产物（本次调研未在其中找到 nav.yaml 等配置文件的副本） |

### 1.2 配置文件清单

```
src/racecar/config/
├── nav.yaml                  # Nav2 导航栈参数（527行，含中文注释）
├── ekf.yaml                  # EKF 里程计融合参数（实际使用版）
├── ekf_carto.yaml            # EKF 里程计融合参数（备选版，含完整英文注释）

│   ├── camera_info.yaml      # 摄像头标定内参
│   ├── params_1.yaml         # usb_cam 参数（/dev/video0, 640x480）
│   └── params_2.yaml         # usb_cam 参数（/dev/video2, 640x480）
└── laser_box/
    └── x4_laser_filter.yaml  # 激光雷达 Box Filter（过滤车体自身点云）
```

## 二、逐个配置文件引用分析

### ✅ `config/nav.yaml` — **有效**

**引用链路：**
```
nav.sh → Run_nav.launch.py (第19-21行)
        → get_package_share_directory('racecar') → 'install/racecar/share/racecar/config/nav.yaml'
        → bringup_launch.py (第46行, params_file参数传入)
        → nav2 全部节点 (amcl, planner, controller, costmap, bt_navigator 等)
```

**证据：** `src/racecar/launch/Run_nav.launch.py` 第 19-21 行：
```python
param_dir = os.path.join(racecar_dir, 'config')
param_file = LaunchConfiguration('params', default=os.path.join(
    param_dir, 'nav.yaml'))
```

**注意：** `bringup_launch.py` 第 119 行有一个回退默认值 `params/nav2_params.yaml`，但该文件不存在于任何目录（`find src install -name nav2_params.yaml` 返回空），且 `Run_nav.launch.py` 始终会传入实际 `params_file` 参数覆盖它，因此不影响。

**结论：** ✅ Nav2 运行时实际加载。**修改后必须同步 `${install}/share/racecar/config/nav.yaml` 并重启 nav.sh**

---

### ✅ `config/ekf.yaml` — **有效**

**引用链路：**
```
nav.sh → Run_car.launch.py (第24行)
       → ekf.launch.py (第15行)
       → ekf_filter_node (robot_localization EKF)
```
```
nav_one.sh → Run_car_one.launch.py (第24行)
           → ekf.launch.py (第15行)
           → ekf_filter_node
```

**证据：** `src/racecar/launch/ekf.launch.py` 第 15、37 行：
```python
ekf_config = Path(get_package_share_directory('racecar'), 'config', 'ekf.yaml')
...
Node(
    condition=UnlessCondition(carto_slam),
    package='robot_localization',
    executable='ekf_node',
    name='ekf_filter_node',
    parameters=[ekf_config],    # ← 这里加载
    ...
)
```

**条件：** 当 `carto_slam=false` 时加载（默认值，正常导航模式）。

**结论：** ✅ EKF 节点运行时实际加载。

---

### ⚠️ `config/ekf_carto.yaml` — **条件有效**

**引用链路：**
```
nav.sh / nav_one.sh / gmapping.sh
  → Run_car.launch.py / Run_car_one.launch.py (第26行)
  → ekf.launch.py (第16、28行)
  → carto_ekf_filter_node (仅当 carto_slam=true 时加载)
```

**证据：** `src/racecar/launch/ekf.launch.py` 第 16、28 行：
```python
ekf_carto_config = Path(get_package_share_directory('racecar'), 'config', 'ekf_carto.yaml')
...
Node(
    condition=IfCondition(carto_slam),
    ...
    parameters=[ekf_carto_config],
)
```

**结论：** ⚠️ 仅在使用 Cartographer SLAM 模式时才加载。当前所有启动脚本（nav.sh、nav_one.sh、gmapping.sh）均使用 `carto_slam=false`，因此**此文件在当前工作流中未被加载**，属于非活跃配置。若未来启用 Cartographer 建图则有效。

### ❌ `config/camera/camera_info.yaml` — **死代码**

**搜索范围：** 全部 src/ 目录下的 `.py` / `.cpp` / `.hpp` / `.h` / `.launch` / `.xml` 文件

**引用结果：** 无任何代码引用此文件。

**核查说明：**
- `camer.launch.py` 使用**内联参数**启动 `usb_cam`，未引用任何外部 yaml 文件
- `xunxian.launch.py` 仅包含 camera launch + line_follow 节点，不涉及 camera_info
- 各 Python 脚本（zhuitong.py、line_follow.py、camer.py、detct.py）中未发现引用此文件

**结论：** ❌ 死代码，可删除或归档。

---

### ❌ `config/camera/params_1.yaml` — **死代码**

**引用结果：** 无任何代码引用此文件。

**核查说明：** 此文件为 `/dev/video0` 摄像头参数配置（640x480, mjpeg2rgb），但 `camer.launch.py` 已用内联参数定义了相同的配置项。该文件从未被任何 launch 文件或节点加载。

**结论：** ❌ 死代码，可删除或归档。

---

### ❌ `config/camera/params_2.yaml` — **死代码**

**引用结果：** 无任何代码引用此文件。

**核查说明：** 此文件为 `/dev/video2`（第二个摄像头）的配置，但系统中 `camer.launch.py` 仅使用 `/dev/video0`，且无其他 launch 文件引用此配置。

**结论：** ❌ 死代码，可删除或归档。

---

### ❌ `config/laser_box/x4_laser_filter.yaml` — **死代码**

**引用结果：** 无任何代码引用此文件。

**核查说明：**
- 全项目源代码中无任何对 `laser_filter`、`x4_laser`、`laser_box`、`box_filter`、`LaserScanBoxFilter` 的引用
- 该文件定义了一个 `LaserScanBoxFilter`，用于过滤 `base_footprint` 坐标系下 `[-0.35, 0.05] × [-0.2, 0.2]` 矩形范围内的点云（即过滤车体自身），但从未被加载到激光雷达管线的任何环节
- `lslidar_driver` 的 launch 文件 `lslidar_launch.py` 不加载任何 filter 配置

**结论：** ❌ 死代码，可删除或归档。

## 三、幽灵目录问题：`install/racecar/share/config/`

**问题：** `install/racecar/share/` 目录下同时存在两个 config 目录：

```
install/racecar/share/
├── config/                ← ❌ 幽灵目录（7 个旧版文件）
│   ├── nav.yaml (349行, 无中文注释)
│   ├── ekf.yaml
│   ├── ekf_carto.yaml
│   └── camera/ ...
│
└── racecar/config/        ← ✅ 实际加载目录（7 个新版文件）
    ├── nav.yaml (527行, 含中文注释)
    ├── ekf.yaml
    ├── ekf_carto.yaml
    └── camera/ ...
```

**根因：** `install/racecar/share/config/` 的所有文件内容与 `install/racecar/share/racecar/config/` 不同，且版本更旧（nav.yaml 仅 349 行 vs 527 行）。这可能是早期 `colcon build` 使用了不同的 `CMakeLists.txt` 配置（如错误配置了 `INSTALL_DIR`）产生的残留。

**影响：** 无运行时影响。所有 launch 文件均通过 `get_package_share_directory('racecar')` 解析路径，指向 `share/racecar/`，不会访问到 `share/config/`。

## 五、建议

### 可立即删除的（死代码，无任何影响）
- `src/racecar/config/camera/` 整个目录（3 个 yaml，全是死代码）
- `src/racecar/config/laser_box/` 整个目录（1 个 yaml，死代码）
- `install/racecar/share/config/` 整个目录（幽灵目录，不被加载且版本落后）

### 建议保留的
- `nav.yaml` — 核心配置，日常调参
- `ekf.yaml` — 核心配置，EKF 融合
- `ekf_carto.yaml` — 建议保留，未来若启用 Cartographer 建图需要用到

### 操作注意事项
1. 删除死代码后无需 `colcon build`，因为 launch 文件本就未引用它们
2. 幽灵目录 `install/racecar/share/config/` 删除后也不影响运行
3. 修改 `nav.yaml` 或 `ekf.yaml` 后仍必须执行 `colcon build --packages-select racecar` 或手动复制到 `install/racecar/share/racecar/config/`，然后重启相关节点
